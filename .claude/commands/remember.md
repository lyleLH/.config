The user's message contains feedback worth saving to memory. Analyze it and create a memory file:

1. Determine whether the feedback is a **correction** (don't do X) or a **confirmation** (yes, keep doing X)
2. Extract the core rule from the user's message
3. Infer the **Why** from conversation context — what happened that triggered this feedback
4. Write a memory file to the current project's memory directory with:
   - `type: feedback` in frontmatter
   - The rule, a **Why:** line, and a **How to apply:** line
5. Update the project's MEMORY.md index

If the user's message is ambiguous about what to remember, ask one clarifying question before saving.
